1. INTRODUCTION                                         *solomon-introduction*

Solomon is a Neovim plugin that deeply integrates Claude Code CLI into your
editor. It provides a hybrid architecture combining terminal embedding with
an MCP bridge for bidirectional communication.

Features:
- Claude Code CLI in a side panel or floating window
- Code actions (explain, improve, task, ask) in normal and visual mode
- Inline replace with animated spinner for improve and task actions
- Streaming response display with configurable keymaps
- MCP server with permessage-deflate compression for full IDE integration
- Live selection tracking broadcast to Claude
- Send files/selections to Claude as @mentions
- Interactive hunk-by-hunk diff review with accept/reject/ask
- Session management with project-scoped picker
- Git integration (diff review, commit messages, blame)
- Auto-refresh with diff highlights when Claude edits files
- Lualine statusline component

4. CONFIGURATION                                       *solomon-configuration*

>lua
  require("solomon").setup({
    terminal = {
      style = "split",
      float_opts = { width = 0.8, height = 0.8, border = "rounded" },
      split_opts = { position = "right", size = 0.4 },
      auto_close = true,
      auto_insert = true,
    },
    keymaps = {
      send = "<leader>aa",
      toggle = "<leader>an",
      focus = "<leader>af",
      close = "<leader>aq",
      ask = "<leader>ak",
      explain = "<leader>ae",
      improve = "<leader>ai",
      task = "<leader>at",
      review = "<leader>aR",
      sessions = "<leader>as",
      continue_session = "<leader>ac",
      diff = "<leader>ad",
      commit = "<leader>am",
      blame = "<leader>ab",
    },
    cli = { cmd = "claude", model = nil, args = {} },
    mcp = { enabled = true, auto_start = true },
  })
<

Set any keymap to `""` to disable it.

5. COMMANDS                                                 *solomon-commands*

:Solomon [subcommand]                                             *:Solomon*

Terminal:~
  `:Solomon`              Open new Claude Code terminal
  `:Solomon open`         Open new Claude Code terminal
  `:Solomon focus`        Focus existing Claude Code terminal
  `:Solomon close`        Close Claude Code terminal

Context:~
  `:Solomon send`         Send selection/file to Claude as @mention

Review:~
  `:Solomon review`       Interactive hunk-by-hunk diff review

Git:~
  `:Solomon diff`         Review unstaged git diff
  `:Solomon diff-staged`  Review staged git diff
  `:Solomon diff-hunk`    Review current git hunk
  `:Solomon commit`       Generate commit message
  `:Solomon blame`        Explain git blame

Sessions:~
  `:Solomon sessions`     Browse project sessions (picker)
  `:Solomon continue`     Continue most recent session
  `:Solomon resume [id]`  Resume session by ID

MCP:~
  `:Solomon mcp-start`    Start the MCP server
  `:Solomon mcp-stop`     Stop the MCP server
  `:Solomon mcp-status`   Show MCP server status
  `:Solomon mcp-log`      Open MCP debug log file

6. KEYMAPS                                                   *solomon-keymaps*

All code actions work in both normal and visual mode.

Code actions:~
  `<leader>ae`   Explain code (popup response)
  `<leader>ai`   Improve code â€” fix, refactor, optimize (inline replace)
  `<leader>at`   Task â€” custom prompt, inline replace
  `<leader>ak`   Ask Claude â€” free-form question (popup response)

Terminal & context:~
  `<leader>an`   Open new Claude Code terminal
  `<leader>af`   Focus existing Claude Code terminal
  `<leader>aq`   Close Claude Code terminal
  `<leader>aa`   Send selection/file to Claude as @mention
  `<leader>ac`   Continue last session
  `<leader>as`   Browse project sessions

Git & review:~
  `<leader>aR`   Interactive diff review (hunk-by-hunk)
  `<leader>ad`   Git diff review
  `<leader>am`   Generate commit message
  `<leader>ab`   Explain git blame

Response window:~
  `q`/`<Esc>`    Close
  `a`            Apply code block (when source available)
  `y`            Yank code block (when source available)
  `d`            Diff view (when source available)

8. REVIEW MODE                                               *solomon-review*

Interactive hunk-by-hunk review of git changes. Trigger with `<leader>aR`
or `:Solomon review`.

Checks in order: unstaged changes, staged changes, unpushed commits.

Review keymaps:~
  `y`   Accept (stage hunk)
  `n`   Reject (revert hunk)
  `o`   Open file at hunk location (exit review)
  `k`   Ask Claude about this hunk (opens prompt)
  `s`   Skip to next hunk
  `q`   Quit review

Status bar shows: `file.lua [3/12] (unstaged)`

9. MCP SERVER                                                   *solomon-mcp*

WebSocket MCP server with permessage-deflate compression. Claude Code
discovers it via lock file at `~/.claude/ide/<port>.lock`.

Tools:~
  `openFile`              Open a file in the editor
  `getOpenEditors`        List all open buffers
  `getCurrentSelection`   Get tracked cursor/visual selection
  `getLatestSelection`    Get most recent tracked selection
  `getDiagnostics`        Read LSP diagnostics
  `openDiff`              Open a diff view
  `closeAllDiffTabs`      Close all diff views
  `getWorkspaceFolders`   Get workspace folders
  `checkDocumentDirty`    Check for unsaved changes
  `saveDocument`          Save a document

Features:~
- Selection tracking via autocmds (debounced, visual mode aware)
- Auto-refresh with checktime polling + diff highlights
- System prompt dynamically built from tool definitions
- Mention queue for pre-connection @mention sends
